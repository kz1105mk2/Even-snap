# EventSnap 要件定義書：Web イベント → Google Calendar 連携

## 1. プロジェクト概要

本プロジェクト名は **EventSnap** とする。Web ページ上に記載されたイベントの開催日時を選択し、右クリック操作から Google Calendar に予定を登録できる Chrome 拡張機能を開発する。ユーザーが日時テキストを選択して右クリックすると、予定作成用のポップアップが表示され、最小限の操作で Google Calendar に予定をブロック（登録）できる。

---

## 2. 機能要件

### 2.1 コンテキストメニュー（右クリックメニュー）

| 項目 | 内容 |
|------|------|
| トリガー | ユーザーが Web ページ上のテキスト（開催日時）を選択し、右クリック |
| メニュー項目 | コンテキストメニューに拡張名を表示（実装では **「EventSnap」**。文言は変更可能） |
| 表示条件 | テキストが選択されている場合のみメニュー項目を表示 |

### 2.2 ポップアップウィンドウ（予定作成フォーム）

コンテキストメニュー選択後に表示されるポップアップウィンドウで、以下の入力項目を提供する。

| 入力項目 | 必須 | 初期値 | 説明 |
|----------|------|--------|------|
| タイトル | × | 選択テキストまたは空欄 | 予定のタイトル |
| 開始日時 | × | 選択テキストから自動パース | 開始の日付と時刻 |
| 終了日時 | × | 開始日時の1時間後（デフォルト） | 終了の日付と時刻 |
| 終日イベント | × | OFF | 終日イベントかどうかのトグル |
| 場所 | × | 空欄 | イベントの開催場所 |
| 説明 | × | 元ページの URL を自動挿入 | イベントの詳細説明 |
| カレンダー選択 | × | デフォルトカレンダー | 登録先の Google Calendar を選択 |
| ステータス | × | 予定あり（busy） | 「予定あり（busy）」/「空き（free）」を選択可能 |
| 公開設定 | × | デフォルト | 「デフォルト」/「公開」/「非公開」を選択可能 |
| 通知設定 | × | カレンダーのデフォルト | 通知タイミングの設定 |

**重要：全ての入力項目は任意入力とし、フォームを開いた状態からそのまま「保存」ボタンを押下するだけで Google Calendar に登録できること。**

#### ポップアップウィンドウの UI 方針

- **デザイン思想**：Jonathan Ive（Apple）のデザイン哲学に倣い、徹底的にミニマルで洗練された UI を実現する
  - 余白を十分に取り、呼吸感のあるレイアウト
  - 装飾を極力排除し、コンテンツとインタラクションに集中させる
  - 色彩は抑制的に使い、アクセントカラーは最小限に
  - タイポグラフィは SF Pro / system-ui を基調とし、視認性と美しさを両立
  - 角丸、シャドウ、トランジションは控えめかつ上品に適用
  - フォーム要素はネイティブに近い自然な操作感を重視
- **サイズ**：全ての入力項目がスクロールなしで一覧・入力できる十分な幅と高さを確保する（目安：幅 480px 以上、高さはコンテンツに応じて自動調整）
- **保存後の動作**：保存成功時はポップアップを自動的に閉じる（成功トーストは不要）

### 2.3 日時の自動パース

- 選択されたテキストから開催日時を自動的に解析し、開始日時フィールドに反映する
- **日本語・英語の両方のフォーマットに対応する**
- 対応フォーマット例：

  **日本語フォーマット：**
  - `2026年3月15日 14:00`
  - `2026/03/15 14:00〜16:00`
  - `3月15日（土）14時〜16時`
  - `令和8年3月15日 14:00`
  - `3/15 14:00-16:00`

  **英語フォーマット：**
  - `March 15, 2026 2:00 PM`
  - `Mar 15, 2026 2:00 PM - 4:00 PM`
  - `2026-03-15T14:00:00`
  - `15 March 2026, 14:00–16:00`
  - `3/15/2026 2pm to 4pm`

- 開始・終了が両方含まれる場合は、それぞれ自動で分離してセットする
- パースに失敗した場合は、フィールドを空欄のままにし、ユーザーが手動入力可能とする

### 2.4 Google Calendar API 連携

| 項目 | 内容 |
|------|------|
| 認証方式 | OAuth 2.0（`chrome.identity` API 経由） |
| 使用 API | Google Calendar API v3 |
| 必要スコープ | `https://www.googleapis.com/auth/calendar` |
| 主要操作 | イベントの作成（`events.insert`） |
| カレンダー一覧取得 | `calendarList.list` でユーザーのカレンダー一覧を取得し、登録先を選択可能にする |
| アカウント | 単一の Google アカウントのみ対応（初期リリース） |

### 2.5 設定画面（オプションページ）

`manifest.json` の `options_page` により **`options/options.html`** を拡張のオプション画面として表示する（ツールバーのポップアップ `popup/popup.html` とは別）。

| 設定項目 | 説明 |
|----------|------|
| デフォルトカレンダー | 予定を登録するデフォルトのカレンダーを選択 |
| デフォルト通知 | デフォルトの通知タイミング（なし / 10分前 / 30分前 / 1時間前 等） |
| デフォルトイベント時間 | 終了日時未指定時のデフォルト所要時間（30分 / 1時間 / 2時間 等） |
| デフォルトステータス | 「予定あり」/「空き」のデフォルト設定 |
| デフォルト公開設定 | 「デフォルト」/「公開」/「非公開」のデフォルト設定 |
| Google アカウント連携 | Google アカウントのログイン / ログアウト管理 |
| OAuth デバッグ情報 | 拡張 ID / Redirect URL / `manifest.json` の `oauth2.client_id` を表示し、設定照合を容易にする |

### 2.6 拡張機能の表示言語

- UI の表示言語は **日本語** とする

## 3. 非機能要件

### 3.1 対応環境

| 項目 | 内容 |
|------|------|
| ブラウザ | Google Chrome（最新安定版） |
| Manifest バージョン | Manifest V3 |
| 対応 OS | Windows / macOS / Linux（Chrome が動作する環境） |

### 3.2 パフォーマンス

- コンテキストメニューの表示は即時（体感遅延なし）
- ポップアップの表示は 500ms 以内
- Google Calendar への登録は 3 秒以内（ネットワーク状態に依存）

### 3.3 セキュリティ

- OAuth トークンは Chrome の `chrome.identity` API で管理（トークンの直接保存は行わない）
- 最小限の権限（パーミッション）のみ要求する
- ユーザーデータは外部サーバーには送信しない（Google API への直接通信のみ）

### 3.4 エラーハンドリング

| エラーケース | 対応 |
|-------------|------|
| 未認証状態での操作 | 認証フローへ誘導するメッセージを表示 |
| OAuth 設定不整合（`bad client id` / `redirect_uri_mismatch`） | 拡張 ID・OAuth クライアント種別・`oauth2.client_id` の不一致を案内し、想定リダイレクト URL を表示 |
| 日時パース失敗 | 手動入力を促す UI を表示（エラーではなく通常フローとして扱う） |
| API エラー（ネットワーク） | リトライ可能なエラーメッセージを表示 |
| API エラー（権限不足） | 再認証を促すメッセージを表示 |
| カレンダー登録成功 | ポップアップを自動的に閉じる |

## 6. 制約事項・前提条件

- ユーザーは Google アカウントを保持していること
- Google Cloud Platform でプロジェクトを作成し、Calendar API を有効化する必要がある（手順は付録 A を参照）
- OAuth クライアント ID（Chrome 拡張機能用）を事前に取得する必要がある（手順は付録 A を参照）
- 日時パースは完全ではなく、特殊なフォーマットには対応できない場合がある
- オフライン環境では Calendar への登録はできない
- 初期リリースでは単一の Google アカウントのみ対応

## 付録 A：Google Cloud Platform セットアップ手順

拡張機能を動作させるために、以下の手順で GCP プロジェクトの作成と OAuth クライアント ID の取得を行う。

### A.0 Google Auth Platform（概要）画面について

コンソール左メニューの **「Google Auth Platform」→「概要」** は、OAuth のリクエスト数・エラー・ユーザー数などをまとめて見るダッシュボードです。クライアント ID の作成・編集は、従来どおり **「API とサービス」→「認証情報」** から行います（[認証情報](https://console.cloud.google.com/apis/credentials)）。

**プロジェクト診断の「安全な OAuth フローの使用」警告**が出ることがあります。これは同意画面やクライアント構成に対する一般的なチェックであり、Chrome 拡張機能が **`manifest.json` の `oauth2` と `chrome.identity.getAuthToken()`** で認証する標準パターンと必ずしも矛盾しません。次を優先して確認してください。

1. OAuth クライアントの種類が **Chrome 拡張機能** で、アイテム ID が拡張 ID と一致しているか（[認証情報](https://console.cloud.google.com/apis/credentials)）
2. `manifest.json` の `oauth2.client_id` が、そのクライアント ID と一致しているか
3. 警告の **「詳細」** リンク先の説明（Google の推奨事項）

**OAuth トークン付与レート**のグラフが空で「データがありません」と出る場合、選択した期間にトークン付与が記録されていないだけのことが多いです。グラフ右側の **Quota limit** はしばしば凡例表示であり、必ずしも上限到達を意味しません。

### A.1 GCP プロジェクトの作成

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセスし、Google アカウントでログイン
2. 画面上部のプロジェクト選択メニューをクリック → **「新しいプロジェクト」** を選択
3. プロジェクト名を入力（例：`eventsnap-dev`）し、**「作成」** をクリック
4. プロジェクトが作成されたら、そのプロジェクトを選択

### A.2 Google Calendar API の有効化

1. 左メニューから **「API とサービス」→「ライブラリ」** を選択
2. 検索バーに **「Google Calendar API」** と入力
3. **「Google Calendar API」** をクリックし、**「有効にする」** ボタンを押す

### A.3 OAuth 同意画面の設定

1. **「API とサービス」→「OAuth 同意画面」** を選択
2. User Type は **「外部」** を選択し、**「作成」** をクリック
3. 必要情報を入力：
   - **アプリ名**：任意（例：`EventSnap`）
   - **ユーザーサポートメール**：自分のメールアドレス
   - **デベロッパーの連絡先情報**：自分のメールアドレス
4. **「スコープを追加または削除」** をクリックし、以下を追加：
   - `https://www.googleapis.com/auth/calendar`
5. テストユーザーに自分の Google アカウントのメールアドレスを追加
6. **「保存して次へ」** で完了

### A.4 OAuth クライアント ID の作成

1. **「API とサービス」→「認証情報」** を選択
2. **「+ 認証情報を作成」→「OAuth クライアント ID」** を選択
3. アプリケーションの種類は **「Chrome 拡張機能」** を選択
4. 名前を入力（例：`EventSnap Extension`）
5. **「アイテム ID」** には拡張機能の ID を入力する
   - ※ 拡張機能をまず Chrome にロードし、`chrome://extensions` から ID を確認して入力
   - ※ **典型ミス:** アイテム ID が別拡張・旧ビルドのまま（例: 以前の `key` なしで読み込んだときの ID）だと、クライアント ID は正しくても `bad client id` になる。**いま `chrome://extensions` に表示されている ID** と GCP のアイテム IDを必ず一致させる（`manifest.json` の `key` を変えると拡張 ID が変わるため、GCP 側も追従が必要）。
6. **「作成」** をクリックし、表示された **クライアント ID** をコピーして控える

> 補足：`OAuth2 request failed: bad client id` が出る場合、(1) クライアント ID の削除・無効化・誤貼り付け、(2) **GCP のアイテム ID と `chrome://extensions` の拡張 ID の不一致**（最も多い）を疑う。

### A.5 拡張機能への設定

1. 取得したクライアント ID を拡張機能の `manifest.json` 内の `oauth2.client_id` に設定する：

```json
{
  "oauth2": {
    "client_id": "YOUR_CLIENT_ID.apps.googleusercontent.com",
    "scopes": [
      "https://www.googleapis.com/auth/calendar"
    ]
  }
}
```

### A.6 拡張機能 ID の確認方法

1. Chrome のアドレスバーに `chrome://extensions` と入力
2. 右上の **「デベロッパーモード」** を ON にする
3. **「パッケージ化されていない拡張機能を読み込む」** をクリックし、拡張機能のディレクトリを選択
4. 読み込まれた拡張機能のカードに表示される **ID** を確認

> **注意**：GCP の OAuth クライアント ID に設定するアイテム ID と、Chrome にロードした拡張機能の ID は一致している必要がある。開発時は ID が固定されるよう `key` フィールドを `manifest.json` に設定することを推奨する。

### A.7 OAuth エラーの確認ポイント（運用時）

1. 拡張機能ポップアップに表示されたエラーメッセージの「想定リダイレクト URL」をコピー
2. `chrome://extensions` で対象拡張の ID を確認
3. GCP の OAuth クライアント（Chrome 拡張機能）で次を照合
   - アイテム ID = 拡張 ID
   - クライアントが有効
4. `manifest.json` の `oauth2.client_id` を最新クライアント ID に更新
5. 拡張機能を再読み込みして再ログイン

GCP でアイテム ID を修正した直後は、反映に **数分〜数時間** かかる場合があります。設定保存後しばらく待ってから試してください。
